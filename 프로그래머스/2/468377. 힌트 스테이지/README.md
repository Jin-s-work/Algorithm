### 문제 설명

<p>당신은 <code>n</code>개의 스테이지(1번 → <code>n</code>번)를 순서대로 모두 해결해야 합니다. 각 스테이지를 클리어 하기 위해서는 해결 비용이 들며, 힌트권을 많이 사용할수록 비용이 줄어듭니다.  </p>

<ul>
<li>힌트권에는 1~<code>n</code>번까지의 번호가 붙어 있습니다. </li>
<li><code>i</code>번 힌트권은 오직 <code>i</code>번 스테이지에서만 사용 가능합니다. </li>
<li>하나의 스테이지에서 사용할 수 있는 힌트권의 최대 개수는 <code>n-1</code>개입니다. </li>
<li>초기에는 힌트권을 가지고 있지 않습니다. </li>
</ul>

<p>마지막 스테이지를 제외한 각 스테이지에서는 해당 스테이지에서 판매하는 힌트 번들을 최대 1개 구매할 수 있습니다. 힌트 번들은 이후 스테이지의 힌트권을 제공하며 비용을 주고 사야합니다.</p>

<ul>
<li>스테이지마다 구매 가능한 힌트 번들의 종류와 판매 가격이 다를 수 있습니다. </li>
<li>힌트 번들에는 힌트권이 총 <code>k</code>장 들어 있으며, 같은 번호의 힌트권이 여러 장 포함될 수도 있습니다. </li>
<li><code>i</code>번 스테이지에서 판매하는 힌트 번들에는 항상 <code>i+1</code>번 이상의 번호를 가진 힌트권만 들어 있습니다. </li>
</ul>

<p>당신은 모든 스테이지를 해결하는데 필요한 최소 비용을 구해야 합니다. </p>

<p>예를 들어 각 스테이지의 해결 비용과 힌트 번들 정보가 아래 표와 같다고 가정해 보겠습니다.</p>

<h5>제한사항</h5>

<ul>
<li>2 ≤ <code>cost</code>의 길이 = <code>n</code> ≤ 16

<ul>
<li><code>cost[i]</code>의 길이 = <code>n</code></li>
<li><code>cost[i][j]</code>는 <code>i+1</code>번 스테이지에서 힌트권을 <code>j</code>개 사용했을 때의 해결 비용입니다.

<ul>
<li>1 ≤ <code>cost[i][j]</code> ≤ 100,000</li>
<li><code>cost[i][j]</code> &gt; <code>cost[i][j+1]</code></li>
</ul></li>
</ul></li>
<li><code>hint</code>의 길이 = <code>n-1</code>

<ul>
<li>2 ≤ <code>hint[i]</code>의 길이 = <code>k+1</code> ≤ 20</li>
<li><code>hint[i][0]</code>은 <code>i+1</code>번 스테이지에서 판매하는 힌트 번들의 판매 가격입니다.

<ul>
<li>0 ≤ <code>hint[i][0]</code> ≤ 1,000,000</li>
</ul></li>
<li><code>hint[i][j]</code>(j&gt;0)는 <code>i+1</code>번 스테이지에서 판매하는 힌트 번들에 <code>j</code>번째로 들어있는 힌트권 번호입니다.

<ul>
<li><code>i+1</code> &lt; <code>hint[i][j]</code> ≤ <code>n</code> (j&gt;0)</li>
</ul></li>
</ul></li>
</ul>
